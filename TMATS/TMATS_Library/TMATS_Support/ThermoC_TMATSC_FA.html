T-MATS: Flow Add (Cantera) Library Block

Purpose

This block is used to add two flows together. When using this block, the secondary flow will be added to the primary flow and the outputs will be determined by using Cantera functions and conserving both mass and enthalpy.


Background

To determine the output flow, this block adds the secondary flow to the primary flow and determines the output parameters by conserving both enthalpy and mass. Output pressure is set to remain equal to the input primary flow pressure. For this block only total values will be populated.

To utilize this block, a Cantera object must exist in the workspace as a global variable named 'fs', which may be created with the Cantera 'importPhase' command. This variable must include all Species defined in the file 'canteraload.m', found at ...\TMATS_Library\MATLAB_Scripts\Cantera_Enabled\+TMATSC.

Example MATLAB setup command:
global fs
fs = importPhase('gri30.cti')


Instructions

To use this block:

  • Ensure that a Cantera object exists in the workspace as a global variable named 'fs' and contains all of the species defined in the file 'canteraload.m'.
    • Note:This can be created with the Cantera 'importPhase' command.
  • Connect the primary and secondary flow inputs to the corresponding place on the block.
  • Connect the outputs to the next blocks in your simulation.

Flow Add (Cantera) Mask Variables

No mask variables are needed for this block, since all of the calculations depend only on the inputs.



Potential Errors

When using this block, an error will occur if a Cantera object does not exist in the workspace as a global variable named 'fs'. An error will also occur if this variable does not contain all of the species defined in 'canteraload.m'.
