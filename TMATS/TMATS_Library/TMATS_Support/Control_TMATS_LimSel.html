T-MATS: Limit Selection Library Block

Purpose

This block can be used to adjust a requested effector value by utilizing limit inputs. The limit inputs can be topping (maximum) or bottoming (minimum) limits.

Background

To compute the Demand Value, this block utilizes a MATALB script that cycles through the input limit values and applies the effects of the limits (maximum or minimum) to the initially entered Request Value.

Instructions

To use this block:

  • Connect the request value and limit value inputs to the corresponding places on the block.
  • Connect the outputs to the next blocks in your simulation.
  • Double click the block and adjust the vector values to account for the input values being topping limits (1) or bottoming limits (-1).



Limit Selection Inputs
Input Description
Request Value Requested Value
Limit Values Values of limiters as a vector. These values may be changed for different conditions (1xn)


Limit Selection Outputs
Output Description
Demand Value Final value after limits are applied
ActiveLimit Value of the active limit. The limit number will coincide with it's location in the Limit Values input vector


Limit Selection Mask Variables
Mask Variable Description
SelectVec_MValues of 1 denote a topping limit and -1 denote a bottoming limit. The input must be in the form of a vector with the same number of values as the Limit Values input vector. (1xn)
LimitsEn_M Enable limits: 1- limits active, 0- limits disabled (1xn)
