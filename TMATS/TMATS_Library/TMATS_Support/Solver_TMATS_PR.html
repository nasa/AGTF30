T-MATS: Parameter Ramp Library Block

Purpose

This block is used to ramp another block's input parameter from an initial value to a final value.


Background

To step the parameter from one value to the next, this block essentially replaces the inital value with a value that is solved for using the specified ramp times. This value varies according to the time at which the block is used:

  • If the block is used before the start ramp time: the value will be set to the initial value.
  • If the block is used after the end ramp time: the value will be set to the final value.
  • If the block is used between the start and end ramp times: the value will be determined via interpolation using the time input and the ramp times.

Note: any parameter changed with this block must be tunable.


Instructions

To use this block:

  • Double click the block.
  • Enter the name of the block that contains the parameter to be adjusted.
  • Enter the name of the parameter to be adjusted.
  • Input the initial value of the parameter.
  • Input the final value of the parameter.
  • Input the start and end ramp time.



Parameter Ramp Inputs
No direct inputs are required for this block, as the needed inputs are specified by the mask variables.

Parameter Ramp Outputs
No direct outputs are required for this block, as the needed outputs are specified by the mask variables.

Parameter Ramp Mask Variables
Mask Variable Description
BN_MBlock name.
Note: The MATLAB command "gcb" can be used to determine a block name.
PN_MParameter name.
Note: The block internal workings makes use of the MATLAB set_parm function and parameter names will need to be in the form required by this function.
IV_MInitial value defined as a string. Size and value must match block's parameter initial size and value.
FV_MFinal value defined as a string. Size must match block's parameter initial size
ST_MStart ramp time defined as a number
ET_MEnd ramp time defined as a number


Potential Errors

When using this block, an error may occur if the parameter names are not specified in the way that the MATLAB function "set_parm" specifies. To resolve this error, enter the parameter names in the form required by this function. Another error may occur if the values input by the user for the initial value and final value mask variables are not the same size as the parameter specified in the block.
